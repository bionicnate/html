AWS Project Summary

Self-Hosted Streaming & Media Management Platform for Deployed Military Personnel

This project will deploy Jellyfin, a self-hosted media streaming platform, along with automated media acquisition and management tools. The system will be distributed across AWS and GCP for redundancy, with Ansible for automation and AWS services for storage, caching, security, and access control

Workflow

Provision VMs

    1. Created 3 AWS t3micro EC2 instances, 3 GCE VMs
      • AWS needs a keypair to connect
      • Modified the security group inbound/outbound to accept TCP:22
        1. later added additional rules as services were deployed
        2. needed to create additional rule for 0.0.0.0/0 to see worker k3s nodes
      • added the AWS key to "authorized_hosts" in GCP for ease of use
      • repeated issues w/ GCE, decided to forgo troubleshooting for the sake of time and focus efforts on AWS
    2. Created additional AWS VM, a t2small to act as control plane
    3. Added Amazon CLI v2
Install K3s & Docker w/Ansible

    1. Ran a simple ping module and checked I could access over SSH
      1. Made a vars file to differentiate master and node for K3s
        1. Specify which hosts with --limit "*master" or --limit "*nodes"
      2. Realized I had to add the .pem to every playbook, so I added a line in ansible.cfg: private_key_file = ~/.ssh/ec2.pem
    2. Grabbed previously created Ansible playbook to install K3s
      1. Ran into an issue where apt did not exist. The AMI used yum as a package manager, whereas my GCP VM had apt.
      2. Plugged the script into Gemini to configure the playbook for conditional package manager between yum and apt.
    3. Eventually gave up and rebuilt VMs to all be the same architecture
    4. Installed K3s on AWS, added kubectl to worker nodes
      1. Had to add EC2 SSH keys and change perms for ease of use
    5. Tried to run a combined playbook for K3s and Docker to deconflict Docker install which accidentally removed all my package manager sources, leaving me unable to install or update
      1. Created a new VM for the K3s master/control plane (3rd time)
    6. Used previous known good Ansible playbook to install Docker
Deploy k3s manangement tool
  1. Installed Portainer on Local
    1. Added an agent to monitor K8s
      1. Connection timeout
    2. Added elastic network IP
      1. Didn't work, had to reboot VM, reinstall K3s, repurposed VM to node
  2. Installed Portainer in Docker on master VM
    1. Wasn't able to connect until after making changes to inbound VPC rules
  3. Added local K8s cluster to monitor
Deploy Manifests for Jellyfin, Radarr, Sonarr, qBittorrent, and Ferdium
  1. Needed to add Python3
    1. Added a virtual environment to the playbook to deploy manifests.
    2. Did not work, so got rid of env entirely and copied manifests to /tmp/
  2. Still had to manually install Python "sudo apt install python3-kubernetes"
  3. Grabbed a compose file from TechHut for Jellyfin Stack, converted it to a manifest
    1. I used the same technique to install as in the Ansible lab, copying the file to the dest /tmp dir, and use kubectl to apply
  4. Same method of installation for the rest of the "arr-stack"
  5. Noticed some pods erroring and evicting, added in some resource limits and that seemed to stop it
    1. kubectl get pods -A
  6. Cleaned up the manifests and combined everything into 2 files, PVC and app deployment
Integrate S3

  6. Stored the key on the master, was able to make a mountpoint to point media services to
